Disabling search restrictions can expose sensible data. This is fine to use in an automatic context like cronjobs but should be avoided for actions triggered by a user.
Check if really needed.

Ask Yourself Whether

  • Data in this request is returned to user?
  • Data is changed after search restrictions are disabled?

Noncompliant Code Example

public void doSomething()
{
    searchRestrictionService.disableSearchRestrictions(); // Noncompliant
    productDAO.find(restrictedProduct);
}

public void doSomething2()
{
    final FlexibleSearchQuery fQuery = new FlexibleSearchQuery("SELECT {passwd} FROM {User}");
    fQuery.setDisableSearchRestrictions(true); // Noncompliant
    flexibleSearchService.searchUnique(fQuery);
}

Recommended Secure Coding Practices

Do not disable search restrictions.
If it is required ensure that no sensible data can be exposed.

See

  • OWASP Top 10 2021 Category A1 - Broken Access Control
  • MITRE, CWE-497 - Exposure of Sensitive System Information
  • MITRE, CWE-200 - Exposure of Sensitive Information to an Unauthorized Actor
